\texttt{evaluate\_generation\_comparison()} feeds the \emph{complete} multi-turn trace and asks for one more turn. For finished tasks, the model correctly outputs text (``Is there anything else?'') but eval counts it as wrong because no reference tool was called.

Evidence:
\begin{itemize}

\subsection{Finding 1: legacy\_cb is completely broken for Fujitsu}
Both \texttt{lcb} runs: 86\% ASR = identical to baseline. 0 improved Fujitsu pairs. But AD resist is 74--81\%, so it partially works on AgentDojo.

\textbf{Root cause:} Dual coefficient schedule. $c_s$ (reroute) = $\alpha_\mathrm{max} \times (1 - (t{-}1)/T)$ decays from 10.0 $\to$ 0.067. $c_r$ (retain) increases from 0 $\to$ 9.93. At step 150, total loss $\approx 0.067 \times \text{reroute} + 9.93 \times \text{retain}$. The last $\sim$50 steps actively undo rerouting. Fujitsu tool-flip (subtle tool swap) is overridden by retain pressure pushing back toward baseline behavior.

\textbf{Why AgentDojo survives:} AD resistance = model outputs text/refusal instead of malicious tool call. This behavioral change is more robust than learning a specific correct tool swap.

\textbf{Verdict:} Not a tuning problem --- structural flaw in decaying schedule with this loss.

\subsection{Finding 2: NTP eval works}
NTP benign\_correct = 20--21\% across all 4 runs (vs 8--12\% FT). Consistent ceiling. NTP eliminates false no\_tool from completed traces: \texttt{lcb} goes 8\% $\to$ 0\%, confirming FT artifacts.

Key insight: under NTP, real degradation signal is in \texttt{no\_tool} rate, not correct rate.

\subsection{Finding 3: Failure modes from trace analysis}
Three distinct modes:
\begin{enumerate}
  \item \textbf{Reasoning text instead of tool call} (\texttt{trip\_pio}, 16/97 broken): model names correct tool but outputs planning text (``I will use get\_total\_spending...'')
  \item \textbf{Code generation instead of tool call} (\texttt{trip\_cfs}, 7/97): model writes Python reimplementing the tool (e.g.\ \texttt{import webbrowser})
  \item \textbf{Direct answer without tool} (\texttt{trip\_cfs}, 2/97): model answers from context (``you are free for lunch at 12:00'')
\end{enumerate}
\texttt{lcb} variants: 1--2/97 broken (near-perfect preservation, but useless for Fujitsu).

\subsection{Finding 4: Triplet\_B collapse paradox}
\texttt{trip\_pio}: Triplet\_B collapses to 0.109. Yet \emph{worst} benign generation (14\% NTP no\_tool).
\texttt{trip\_cfs}: Triplet\_B stays at 0.937. \emph{Best} benign (2\% NTP no\_tool).

Cause: under \texttt{post\_injection\_only}, benign data has all-zero loss mask $\to$ zero gradient from benign examples. Triplet\_B is computed via pooling mask (separate), so it ``looks good'' while actual generation degrades. \textbf{Pooled representation distance does not predict generation quality.}

\section{Data Augmentation: LLM-Based Trace Completion (Mar 5, 2026)}
% ===================================================================

\subsection{Problem: Skeletons Can't Drive Representation Rerouting}

The Fujitsu B4 dataset (13{,}246 traces) consists entirely of \textbf{skeletons}:
system + user messages only, with \emph{no} assistant response.
Under \texttt{injection\_aware} masking, a skeleton produces:

\begin{center}
\begin{tabular}{lcl}
\toprule
Token region & Weight & Role \\
\midrule
System prompt & 0.0 & (always masked) \\
User query (pre-injection) & 0.0 & (trusted input) \\
Injection span & 0.5 & detection signal \\
\bad{Post-injection assistant} & \bad{absent} & \bad{no resistance signal} \\
\bottomrule
\end{tabular}
\end{center}

Without a post-injection assistant response (the 1.0-weighted tokens that teach
\emph{resistance}), the CB loss receives only the weak 0.5 detection signal from
the injection span itself. The model learns to detect injections but never
learns what to do about them. This explains why skeleton-only training with
\texttt{--skeleton-policy cb\_full\_sequence} works at all --- it masks the entire
user message uniformly rather than relying on the absent assistant response.

\subsection{Solution: LLM-Based Skeleton Completion}

We complete each skeleton into a full 5-message multi-turn trace using
vLLM inference with \textbf{the same Llama-3.1-8B-Instruct model we train on}.
This ensures the completions are in the model's native token distribution,
maximizing representation quality for circuit-breaking.

\medskip
\noindent\textbf{Canonical completed trace structure:}

\begin{center}
\small
\begin{tabular}{clcc}
\toprule
Msg\# & Role & injection\_aware & cb\_full (pool) \\
\midrule
0 & \texttt{system}: B4 system prompt & 0.0 & 0.0 \\
1 & \texttt{user}: query + injection & \textbf{0.5} (injection span) & 1.0 \\
2 & \texttt{assistant}: tool call (wrong tool) & \good{\textbf{1.0}} & 1.0 \\
3 & \texttt{tool}: simulated output & 0.0 & 1.0 \\
4 & \texttt{assistant}: follow-up response & \good{\textbf{1.0}} & 1.0 \\
\bottomrule
\end{tabular}
\end{center}

Each skeleton produces \textbf{two} completed traces:

\begin{enumerate}
  \item \textbf{Harmful} (attacked): User message \emph{with} injection $\to$
        model calls wrong tool (\texttt{search\_web} instead of
        \texttt{retrieve\_multimodal\_docs}). The 1.0-weighted assistant tokens
        carry the resistance signal. \emph{Injection span preserved.}
  \item \textbf{Benign} (correct): User message \emph{without} injection
        (clean query only) $\to$ model calls correct tool.
        Injection\_aware mask is \textbf{all-zero} (retain set, no CB penalty).
\end{enumerate}

This contrastive pair structure directly implements the BIPIA insight:
given injected input vs.\ clean input, the correct tool call should be
identical. The harmful trace teaches detection + resistance; the benign
counterpart anchors correct behavior.

\begin{center}
\small

\midrule

\subsection{Why This Should Work (Literature-Informed)}

\begin{enumerate}
  \item \textbf{Circuit Breakers}: Rerouting at layers 10+20 requires
        clear ``injection present $\to$ wrong action taken'' signal in the
        representation. Completed traces provide exactly this: the tool call
        decision token is \emph{after} the injection, in the 1.0-weighted region.

  \item \textbf{Latent Unlearning}: Harmful/benign tool calls share identical
        JSON format (\texttt{\{``name'': ``...'', ``parameters'': \{...\}\}}).
        The injection\_aware mask zeros out this shared format, focusing loss
        only on the differing tool \emph{name} tokens. This avoids the
        forget/retain n-gram overlap that kills unlearning.

  \item \textbf{BIPIA contrastive}: Each skeleton yields a matched pair:
        (injected $\to$ wrong tool) vs.\ (clean $\to$ correct tool). The
        \emph{only} difference is the injection in the user message.
        Contrastive training isolates the causal effect of the injection.

  \item \textbf{StruQ structural separation}: The completed trace has
        clear channel boundaries: system (trusted), user (partially trusted),
        injection (untrusted), assistant (decision point). The mask weights
        directly encode these trust levels (0.0 / 0.5 / 1.0).

  \item \textbf{Same-model completions}: Using vLLM with the same
        Llama-3.1-8B-Instruct we train on means the tool call tokens are in
        the model's own distribution. The representations at layers 10+20
        are native, not cross-model artifacts. This should make rerouting
        more effective than training on Claude/GPT-generated traces.
\end{enumerate}

\section{Trace Format Analysis: What Is Optimal for Circuit-Breaking?}
% ===================================================================

We survey all 10 reference papers to determine the most effective training
data format for representation rerouting on agentic tool-calling traces.
This is a first-principles analysis: what format makes the harmful/benign
separation \emph{easiest} to learn at the representation level, while
remaining deployment-compatible for real agents?

\subsection{How Each Paper Formats Training Data}

\begin{center}
\small
\begin{tabular}{p{3cm}p{1.5cm}p{2.5cm}p{5.5cm}}
\toprule
Paper & Turns & Loss target & Format \\
\midrule
Circuit Breakers (Zou) & 1 & All tokens (user+asst) & Chat template; 50\% have empty user prompt. 512+512 fixed. \\
Instruction Hierarchy (OpenAI) & Multi & SFT on response & System$\to$user$\to$asst(tool\_call)$\to$tool(+inj)$\to$asst. Context distillation. \\
BIPIA (Microsoft) & 1 & NTP on response & \texttt{<data>}...\texttt{</data>} boundary tokens around external content. \\
StruQ (Berkeley) & 1 & NTP on response & 5 reserved tokens: \texttt{[MARK]}, \texttt{[INST]}, \texttt{[INPT]}, \texttt{[RESP]}, \texttt{[COLN]}. \\
Latent Unlearning & N/A & MSE on \emph{averaged} hidden states & Raw text passages. Forget set vs.\ retain set from different domains. \\
AgentFlux & Multi & Separate tool-name vs.\ argument masks & Multi-turn trajectories. Two LoRA adapters with different loss masks. \\

\end{tabular}
\end{center}

\subsection{Six Key Findings}

\paragraph{1. CB paper uses ALL tokens for rerouting, not just assistant.}
Zou et al.\ compute their ReLU(cos\_sim) loss over \emph{every} non-padding
token, including the user prompt. The attention mask IS the loss mask.
Quote: ``For enhanced robustness, we apply the circuit-breaking loss to both
the user and assistant text within the circuit breaker set.'' Furthermore,
50\% of Llama-3 samples have an \textbf{empty user prompt}---only the harmful
assistant text---to teach rerouting even without a harmful prompt preceding it.

\medskip\noindent
\textbf{Implication for us:} Our \texttt{injection\_aware} policy uses
\emph{differential} weights (0.0/0.5/1.0) rather than binary all-or-nothing.
This is strictly more expressive than the original CB approach. The original
CB paper never had to distinguish injection from legitimate content because
their use case was text-only harmful completions, not agentic injection.
Our 33.7\% ASR result (vs.\ 61\% with \texttt{cb\_full\_sequence}) validates
that token-level granularity matters for the agentic setting.

\paragraph{2. Structural position is what models learn for injection defense.}
Three independent papers converge on this:
\begin{itemize}
  \item \textbf{Instruction Hierarchy}: Models learn that tool-output content
        is lowest privilege. Training on injection \emph{in the tool output
        position} generalizes to unseen tools and attack types.
  \item \textbf{BIPIA}: ``Removal of boundary awareness has a greater impact
        on ASR than removal of explicit reminder.'' The \texttt{<data>}/\texttt{</data>}
        tokens matter more than ``don't follow injected instructions.''
  \item \textbf{StruQ}: Reserved tokens for channel boundaries ($<$2\% ASR
        on manual attacks). Front-end filters ensure untrusted data cannot
        contain delimiter strings.
\end{itemize}

\noindent\textbf{Implication for us:} The Llama 3.1 chat template
\emph{already encodes} a natural privilege hierarchy through its role headers:
\texttt{<|start\_header\_id|>system<|end\_header\_id|>} (highest) $>$
\texttt{user} $>$ \texttt{tool} (lowest) $>$
\texttt{assistant} (model's decision).
We do not need custom boundary tokens (BIPIA) or reserved delimiters (StruQ).
The native template is the boundary marker. This is optimal because:
(a)~the model already has pre-trained awareness of these headers from
instruction tuning, and (b)~at inference time the agent framework sends
messages in exactly this format---zero distribution shift.

\paragraph{3. Forget/retain sets must be representationally distinct.}
Latent Unlearning's critical finding: when forget and retain distributions
share $>$20\% unigram overlap, representation-level unlearning \emph{fails
entirely}. Our harmful and benign tool calls share \emph{identical} JSON
format (\texttt{\{``name'': ``search\_web'', ``parameters'': ...\}}).

\medskip\noindent
\textbf{Implication:} This is exactly why \texttt{injection\_aware} works
and \texttt{cb\_full\_sequence} is suboptimal. \texttt{cb\_full\_sequence}
puts weight 1.0 on both the injection text AND the shared-format tool call
JSON. The model cannot separate ``harmful tool call'' from ``benign tool call''
when the surface tokens are nearly identical. \texttt{injection\_aware}
solves this by putting 0.5 weight on the injection span (which is
representationally distinct---HTML comments vs.\ clean queries) and 1.0
weight on post-injection assistant tokens (where the \emph{decision} diverges,
not the format).

\paragraph{4. Tool selection and argument generation are separate tasks.}
AgentFlux's DualTune finding: separating tool-name loss from argument loss
yields +22.5pp over joint fine-tuning. Hammer's finding: masking function
names to random strings forces description-based reasoning and improves
cross-benchmark generalization.

\medskip\noindent
\textbf{Implication:} For circuit-breaking, we primarily care about the
tool \emph{selection} decision (calling \texttt{search\_web} instead of
\texttt{retrieve\_multimodal\_docs}), not the arguments. Our
\texttt{tool\_name\_weighted} policy (implemented but untested in sweep)
captures exactly this insight: weight 1.0 on tool name tokens, 0.3 on
arguments. This could be combined with \texttt{injection\_aware} in future.

\paragraph{5. Multi-turn agentic format is necessary for deployment.}
The Instruction Hierarchy paper showed that training on injection in the
tool-output position of a multi-turn trace \emph{generalizes} to unseen
tools. Single-turn format (CB paper's default) loses the critical
\texttt{tool}$\to$\texttt{assistant} causal chain that makes agent injection
dangerous. At inference, the agent framework sends multi-turn messages;
training on the same format eliminates distribution shift.

\paragraph{6. Full-sequence loss is directionally better for agents.}
The prompt masking blog found +3pp pass rate when computing loss over the
full sequence (including prior tool-observation pairs) vs.\ completion-only.
Not statistically significant (p=0.10), but directionally consistent with
the CB paper's choice to apply loss to all tokens. Our \texttt{injection\_aware}
is a middle ground: it zeros out system/tool tokens but includes user
(injection) and assistant tokens---more selective than full-sequence, more
inclusive than completion-only.

\subsection{Conclusion: The Native Llama 3.1 Agentic Format Is Optimal}

No paper proposes a format that dominates for all three of our requirements
(representation rerouting + injection defense + tool-calling preservation).
Synthesizing the evidence, the optimal format is:

\begin{center}
\fbox{\parbox{0.92\textwidth}{
\textbf{Format:} Native Llama 3.1 multi-turn chat template with
\texttt{<|python\_tag|>} tool calls. \\
\textbf{Loss:} \texttt{injection\_aware} (0.5 on injection, 1.0 on
post-injection assistant, 0.0 elsewhere). \\
\textbf{Pool:} \texttt{cb\_full\_sequence} (broad representation sampling). \\
\textbf{Turns:} 5 messages (system $\to$ user $\to$ assistant[tool\_call]
$\to$ tool $\to$ assistant[follow-up]).
}}
\end{center}

\noindent
The exact rendered format for a harmful completed trace:

\begin{verbatim}
<|begin_of_text|><|start_header_id|>system<|end_header_id|>
                                                     w=0.0
{system prompt with tool schemas}<|eot_id|>
                                                     w=0.0
<|start_header_id|>user<|end_header_id|>
                                                     w=0.0
{clean query}                                        w=0.0
{injection text}                                     w=0.5
<|eot_id|>
<|start_header_id|>assistant<|end_header_id|>
                                                     w=1.0
<|python_tag|>{"name":"search_web","parameters":{..}}
                                                     w=1.0
<|eom_id|>
<|start_header_id|>tool<|end_header_id|>
                                                     w=0.0
{tool output}<|eot_id|>
                                                     w=0.0
<|start_header_id|>assistant<|end_header_id|>
                                                     w=1.0
{follow-up response}<|eot_id|>                       w=1.0
\end{verbatim}

\noindent This is optimal because:

\begin{enumerate}
  \item \textbf{Native token distribution.} The same model generates these
        tokens during vLLM completion, so layer 10+20 representations are
        in-distribution. No cross-model artifacts (BIPIA used GPT-4, CB used
        an uncensored LLM).

  \item \textbf{Built-in privilege hierarchy.} The role headers
        (\texttt{system} $>$ \texttt{user} $>$ \texttt{tool}) serve the same
        function as BIPIA's \texttt{<data>} tokens and StruQ's
        \texttt{[MARK]} delimiters, but they are already in the model's
        vocabulary and pre-trained distribution.

  \item \textbf{Representational distinctness where it matters.} The 0.5
        weight goes on injection tokens (HTML comments, \texttt{<INFORMATION>}
        tags), which are surface-form distinct from clean queries. The 1.0
        weight goes on assistant decision tokens, where harmful (wrong tool)
        and benign (correct tool) diverge at the \emph{tool name} tokens.
        This addresses the n-gram overlap problem from latent unlearning.

  \item \textbf{Deployment-compatible.} At inference, the agent framework
        (LangChain, vLLM tool mode, etc.) sends messages in exactly this
        Llama 3.1 chat format. The representations the model sees during
        inference are identical to training. No format adaptation needed.

  \item \textbf{The \texttt{<|python\_tag|>} is a natural circuit-breaker
        target.} This special token precedes every tool call. In harmful
        traces, the layer 10+20 representation at this token position
        encodes ``I am about to call a tool that the injection told me to
        call.'' Rerouting this representation is precisely what CB needs.
\end{enumerate}

\noindent\textbf{What we considered and rejected:}

\begin{center}
\small
\begin{tabular}{p{4cm}p{9cm}}
\toprule
Alternative & Why rejected \\
\midrule
Custom boundary tokens (BIPIA, StruQ) & Adds tokens not in pre-trained vocabulary. StruQ showed initialization matters critically. Llama's role headers already serve this purpose. \\

drove the improvement, but the pooling still sampled from the full skeleton

\emph{resistance} tokens to reroute.

\subsection{Pipeline Simplification}

Instead of the 5-operation LLM augmentation pipeline (requiring vLLM on cluster),
we use two local preprocessing steps:

\begin{enumerate}
  \item \textbf{Complete Fujitsu} (\texttt{augment\_llm.py complete\_fujitsu --dry-run}):
        Each skeleton $\to$ 2 completed traces (harmful + benign contrastive pair).
        Dry-run mode uses template tool calls (no vLLM). Content doesn't matter for CB ---
        position and mask weights do.
  \item \textbf{Truncate AgentDojo} (\texttt{truncate\_to\_injection\_window.py}):
        Cuts multi-turn traces to the injection decision window. For harmful:
        keep messages[0..first\_post\_injection\_assistant]. For benign: keep
        first tool$\to$assistant cycle. Drops 225 harmful traces without injection spans.
\end{enumerate}
