\subsection{Benign Eval Metric Bug}
\texttt{evaluate\_generation\_comparison()} feeds the \emph{complete} multi-turn trace and asks for one more turn. For finished tasks, the model correctly outputs text (``Is there anything else?'') but eval counts it as wrong because no reference tool was called.

Evidence:
\begin{itemize}
  \item Baseline correct\_tool = 2--6\% (model itself barely ``matches'')
  \item \texttt{get\_current\_year} dominates broken examples --- but this tool appears in \emph{zero} trace files. The baseline model hallucinates it.
  \item ``Broken'' examples show CB correctly NOT hallucinating unnecessary tool calls
\end{itemize}

Fix: \texttt{next\_tool\_prediction} mode --- truncate trace before last tool-calling assistant message, check if model predicts correct next tool.

\subsection{Training Dynamics: l10\_20 vs l20-31}
\begin{itemize}
  \item l10\_20 (\texttt{post\_injection}): Loss 0.52 $\to$ 0.26. Triplet\_B stays $\sim$0.11. Learning continues.
  \item l20-31 (\texttt{post\_injection}): Loss 0.51 $\to$ 0.10. \bad{Triplet\_B crashes to 0.0 by step 90}. Benign reps converged to frozen (no gradient signal). 12 layers of LoRA disrupts intermediate computation.
  \item Matches Latent Unlearning prediction: deeper layers have lower noise sensitivity.
\end{itemize}

\subsection{Finding 1: legacy\_cb is completely broken for Fujitsu}
Both \texttt{lcb} runs: 86\% ASR = identical to baseline. 0 improved Fujitsu pairs. But AD resist is 74--81\%, so it partially works on AgentDojo.

\textbf{Root cause:} Dual coefficient schedule. $c_s$ (reroute) = $\alpha_\mathrm{max} \times (1 - (t{-}1)/T)$ decays from 10.0 $\to$ 0.067. $c_r$ (retain) increases from 0 $\to$ 9.93. At step 150, total loss $\approx 0.067 \times \text{reroute} + 9.93 \times \text{retain}$. The last $\sim$50 steps actively undo rerouting. Fujitsu tool-flip (subtle tool swap) is overridden by retain pressure pushing back toward baseline behavior.

\textbf{Why AgentDojo survives:} AD resistance = model outputs text/refusal instead of malicious tool call. This behavioral change is more robust than learning a specific correct tool swap.

\textbf{Verdict:} Not a tuning problem --- structural flaw in decaying schedule with this loss.

\subsection{Finding 2: NTP eval works}
NTP benign\_correct = 20--21\% across all 4 runs (vs 8--12\% FT). Consistent ceiling. NTP eliminates false no\_tool from completed traces: \texttt{lcb} goes 8\% $\to$ 0\%, confirming FT artifacts.

Key insight: under NTP, real degradation signal is in \texttt{no\_tool} rate, not correct rate.

\subsection{Finding 3: Failure modes from trace analysis}
Three distinct modes:
\begin{enumerate}
  \item \textbf{Reasoning text instead of tool call} (\texttt{trip\_pio}, 16/97 broken): model names correct tool but outputs planning text (``I will use get\_total\_spending...'')
  \item \textbf{Code generation instead of tool call} (\texttt{trip\_cfs}, 7/97): model writes Python reimplementing the tool (e.g.\ \texttt{import webbrowser})
  \item \textbf{Direct answer without tool} (\texttt{trip\_cfs}, 2/97): model answers from context (``you are free for lunch at 12:00'')
\end{enumerate}
\texttt{lcb} variants: 1--2/97 broken (near-perfect preservation, but useless for Fujitsu).

\subsection{Finding 4: Triplet\_B collapse paradox}
\texttt{trip\_pio}: Triplet\_B collapses to 0.109. Yet \emph{worst} benign generation (14\% NTP no\_tool).
\texttt{trip\_cfs}: Triplet\_B stays at 0.937. \emph{Best} benign (2\% NTP no\_tool).

Cause: under \texttt{post\_injection\_only}, benign data has all-zero loss mask $\to$ zero gradient from benign examples. Triplet\_B is computed via pooling mask (separate), so it ``looks good'' while actual generation degrades. \textbf{Pooled representation distance does not predict generation quality.}

\subsection{Finding 5: Baseline NTP ceiling is 21\% (ANSWERED)}
Baseline NTP benign: 20--21\% correct, 0\% no\_tool across all 4 runs. \textbf{This is the ceiling.} The remaining $\sim$79\% are wrong-tool predictions by the base model itself.

CB model degradation: \texttt{trip\_cfs} $-$2pp correct, $+$3pp no\_tool. \texttt{trip\_pio} 0pp correct, $+$12pp no\_tool. \texttt{lcb} identical to baseline (learns nothing).

\subsection{Finding 6: legacy\_cb gradient failure (deeper root cause)}
Reroute component at every checkpoint:
\begin{verbatim}
[Step 0]   cos_sim=1.0000  grad_norm=0.0007  (cs=10, cr=0)
[Step 50]  cos_sim=1.0000  grad_norm=3.6     (retain gradient)
[Step 100] cos_sim=1.0000  grad_norm=2.5     (retain gradient)
\end{verbatim}

Even at step 0 with pure reroute (cs=10, cr=0), grad\_norm$\approx$0. \texttt{ReLU(cos\_sim)} provides near-zero gradient for LoRA-scale perturbations (rank 16 in 4096-dim). The loss surface is flat around cos\_sim$=$1.0 at LoRA scale. \textbf{Not just a schedule problem --- a loss/LoRA incompatibility.} Original CB paper uses full fine-tuning.

\section{Conclusions (Updated Mar 4)}
% ===================================================================

\textbf{Best config:} \texttt{trip\_cfs\_l10\_20} (triplet\_full + cb\_full\_sequence + layers 10,20). Fujitsu ASR 60--63\%, AD resist 82--83\%, NTP benign 19--21\% correct / 2--3\% no\_tool. Zero gibberish. \textbf{Reproduced across 2 independent runs.}

\textbf{Settled decisions:}
\begin{itemize}
  \item Layers: 10,20 (not 20--31)
  \item Steps: 150
  \item Pooling: correct mode, $d_\mathrm{cos}$, margins 1.2/1.6
  \item Loss: triplet\_full (not legacy\_cb --- incompatible with LoRA)
  \item Policy: cb\_full\_sequence (post\_injection\_only creates harmful tradeoff)
  \item Eval: next\_tool\_prediction mode for benign metrics
\end{itemize}

\textbf{Answered questions:}
\begin{itemize}
  \item \good{21\% NTP benign correct IS the baseline ceiling.} CB degradation: $-$2pp correct, $+$3pp no\_tool.
  \item \good{legacy\_cb fails due to loss function + LoRA incompatibility}, not just schedule.
  \item \good{trip\_pio creates harmful tradeoff}: $+$3pp AD malicious, $+$12pp benign no\_tool.
\end{itemize}

\textbf{Remaining levers:}
\begin{itemize}
  \item $\gamma_\mathrm{kl}$ increase (0.1 $\to$ 0.3--0.5): reduce ``code gen instead of tool call'' failure mode
  \item Restrict LoRA to layers 0--20 (original CB paper)
  \item More steps: Trip\_H=1.015 at step 150, margin=1.6 --- rerouting hasn't fully converged
  \item Training data diversity (AgentHarm, WebArena, etc.)
\end{itemize}

\section{Sweep 6 Plan: Balanced Training (Mar 5, 2026)}

\textbf{Goal:} Eliminate remaining 33/98 Fujitsu failures by addressing training data imbalance
and providing stronger rerouting signal.
