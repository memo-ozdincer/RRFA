\texttt{evaluate\_generation\_comparison()} feeds the \emph{complete} multi-turn trace and asks for one more turn. For finished tasks, the model correctly outputs text (``Is there anything else?'') but eval counts it as wrong because no reference tool was called.

Evidence:

\subsection{Finding 1: legacy\_cb is completely broken for Fujitsu}
Both \texttt{lcb} runs: 86\% ASR = identical to baseline. 0 improved Fujitsu pairs. But AD resist is 74--81\%, so it partially works on AgentDojo.

\textbf{Root cause:} Dual coefficient schedule. $c_s$ (reroute) = $\alpha_\mathrm{max} \times (1 - (t{-}1)/T)$ decays from 10.0 $\to$ 0.067. $c_r$ (retain) increases from 0 $\to$ 9.93. At step 150, total loss $\approx 0.067 \times \text{reroute} + 9.93 \times \text{retain}$. The last $\sim$50 steps actively undo rerouting. Fujitsu tool-flip (subtle tool swap) is overridden by retain pressure pushing back toward baseline behavior.

\textbf{Why AgentDojo survives:} AD resistance = model outputs text/refusal instead of malicious tool call. This behavioral change is more robust than learning a specific correct tool swap.

\textbf{Verdict:} Not a tuning problem --- structural flaw in decaying schedule with this loss.

\subsection{Finding 2: NTP eval works}
NTP benign\_correct = 20--21\% across all 4 runs (vs 8--12\% FT). Consistent ceiling. NTP eliminates false no\_tool from completed traces: \texttt{lcb} goes 8\% $\to$ 0\%, confirming FT artifacts.

Key insight: under NTP, real degradation signal is in \texttt{no\_tool} rate, not correct rate.

\subsection{Finding 3: Failure modes from trace analysis}
Three distinct modes:
\begin{enumerate}
  \item \textbf{Reasoning text instead of tool call} (\texttt{trip\_pio}, 16/97 broken): model names correct tool but outputs planning text (``I will use get\_total\_spending...'')
  \item \textbf{Code generation instead of tool call} (\texttt{trip\_cfs}, 7/97): model writes Python reimplementing the tool (e.g.\ \texttt{import webbrowser})
  \item \textbf{Direct answer without tool} (\texttt{trip\_cfs}, 2/97): model answers from context (``you are free for lunch at 12:00'')
\end{enumerate}
\texttt{lcb} variants: 1--2/97 broken (near-perfect preservation, but useless for Fujitsu).

\subsection{Finding 4: Triplet\_B collapse paradox}
\texttt{trip\_pio}: Triplet\_B collapses to 0.109. Yet \emph{worst} benign generation (14\% NTP no\_tool).
\texttt{trip\_cfs}: Triplet\_B stays at 0.937. \emph{Best} benign (2\% NTP no\_tool).

Cause: under \texttt{post\_injection\_only}, benign data has all-zero loss mask $\to$ zero gradient from benign examples. Triplet\_B is computed via pooling mask (separate), so it ``looks good'' while actual generation degrades. \textbf{Pooled representation distance does not predict generation quality.}

\section{Data Augmentation: LLM-Based Trace Completion (Mar 5, 2026)}

\subsection{Problem: Skeletons Can't Drive Representation Rerouting}

The Fujitsu B4 dataset (13{,}246 traces) consists entirely of \textbf{skeletons}:
system + user messages only, with \emph{no} assistant response.
Under \texttt{injection\_aware} masking, a skeleton produces:

\begin{center}
\begin{tabular}{lcl}
\toprule
Token region & Weight & Role \\
\midrule
System prompt & 0.0 & (always masked) \\
User query (pre-injection) & 0.0 & (trusted input) \\
Injection span & 0.5 & detection signal \\
\bad{Post-injection assistant} & \bad{absent} & \bad{no resistance signal} \\
\bottomrule
\end{tabular}
\end{center}

Without a post-injection assistant response (the 1.0-weighted tokens that teach
\emph{resistance}), the CB loss receives only the weak 0.5 detection signal from
the injection span itself. The model learns to detect injections but never
learns what to do about them. This explains why skeleton-only training with
\texttt{--skeleton-policy cb\_full\_sequence} works at all --- it masks the entire
user message uniformly rather than relying on the absent assistant response.

\subsection{Solution: LLM-Based Skeleton Completion}

We complete each skeleton into a full 5-message multi-turn trace using
vLLM inference with \textbf{the same Llama-3.1-8B-Instruct model we train on}.
This ensures the completions are in the model's native token distribution,
maximizing representation quality for circuit-breaking.

\medskip
\noindent\textbf{Canonical completed trace structure:}

\begin{center}
\small
\begin{tabular}{clcc}
\toprule
Msg\# & Role & injection\_aware & cb\_full (pool) \\
\midrule
0 & \texttt{system}: B4 system prompt & 0.0 & 0.0 \\
1 & \texttt{user}: query + injection & \textbf{0.5} (injection span) & 1.0 \\
2 & \texttt{assistant}: tool call (wrong tool) & \good{\textbf{1.0}} & 1.0 \\
3 & \texttt{tool}: simulated output & 0.0 & 1.0 \\
4 & \texttt{assistant}: follow-up response & \good{\textbf{1.0}} & 1.0 \\
\bottomrule
\end{tabular}
\end{center}

Each skeleton produces \textbf{two} completed traces:

\begin{enumerate}
  \item \textbf{Harmful} (attacked): User message \emph{with} injection $\to$
        model calls wrong tool (\texttt{search\_web} instead of
        \texttt{retrieve\_multimodal\_docs}). The 1.0-weighted assistant tokens
        carry the resistance signal. \emph{Injection span preserved.}
  \item \textbf{Benign} (correct): User message \emph{without} injection
        (clean query only) $\to$ model calls correct tool.
        Injection\_aware mask is \textbf{all-zero} (retain set, no CB penalty).
\end{enumerate}

This contrastive pair structure directly implements the BIPIA insight:
given injected input vs.\ clean input, the correct tool call should be
identical. The harmful trace teaches detection + resistance; the benign
counterpart anchors correct behavior.

\subsection{Why This Should Work (Literature-Informed)}

\begin{enumerate}
  \item \textbf{Circuit Breakers}: Rerouting at layers 10+20 requires
        clear ``injection present $\to$ wrong action taken'' signal in the
        representation. Completed traces provide exactly this: the tool call
        decision token is \emph{after} the injection, in the 1.0-weighted region.

  \item \textbf{Latent Unlearning}: Harmful/benign tool calls share identical
        JSON format (\texttt{\{``name'': ``...'', ``parameters'': \{...\}\}}).
        The injection\_aware mask zeros out this shared format, focusing loss
        only on the differing tool \emph{name} tokens. This avoids the
        forget/retain n-gram overlap that kills unlearning.

  \item \textbf{BIPIA contrastive}: Each skeleton yields a matched pair:
        (injected $\to$ wrong tool) vs.\ (clean $\to$ correct tool). The
        \emph{only} difference is the injection in the user message.
        Contrastive training isolates the causal effect of the injection.

  \item \textbf{StruQ structural separation}: The completed trace has
        clear channel boundaries: system (trusted), user (partially trusted),
        injection (untrusted), assistant (decision point). The mask weights
        directly encode these trust levels (0.0 / 0.5 / 1.0).

  \item \textbf{Same-model completions}: Using vLLM with the same
        Llama-3.1-8B-Instruct we train on means the tool call tokens are in
        the model's own distribution. The representations at layers 10+20
        are native, not cross-model artifacts. This should make rerouting
        more effective than training on Claude/GPT-generated traces.
\end{enumerate}

\section{Trace Format Analysis: What Is Optimal for Circuit-Breaking?}
% ===================================================================

We survey all 10 reference papers to determine the most effective training
data format for representation rerouting on agentic tool-calling traces.
This is a first-principles analysis: what format makes the harmful/benign
separation \emph{easiest} to learn at the representation level, while
remaining deployment-compatible for real agents?

\subsection{How Each Paper Formats Training Data}

\subsection{Six Key Findings}

\paragraph{1. CB paper uses ALL tokens for rerouting, not just assistant.}
Zou et al.\ compute their ReLU(cos\_sim) loss over \emph{every} non-padding
token, including the user prompt. The attention mask IS the loss mask.
Quote: ``For enhanced robustness, we apply the circuit-breaking loss to both
the user and assistant text within the circuit breaker set.'' Furthermore,
50\% of Llama-3 samples have an \textbf{empty user prompt}---only the harmful
assistant text---to teach rerouting even without a harmful prompt preceding it.

\medskip\noindent
\textbf{Implication for us:} Our \texttt{injection\_aware} policy uses
\emph{differential} weights (0.0/0.5/1.0) rather than binary all-or-nothing.
This is strictly more expressive than the original CB approach. The original
CB paper never had to distinguish injection from legitimate content because
their use case was text-only harmful completions, not agentic injection.
Our 33.7\% ASR result (vs.\ 61\% with \texttt{cb\_full\_sequence}) validates
that token-level granularity matters for the agentic setting.

\paragraph{2. Structural position is what models learn for injection defense.}
Three independent papers converge on this:
\begin{itemize}
  \item \textbf{Instruction Hierarchy}: Models learn that tool-output content
        is lowest privilege. Training on injection \emph{in the tool output
        position} generalizes to unseen tools and attack types.
  \item \textbf{BIPIA}: ``Removal of boundary awareness has a greater impact
        on ASR than removal of explicit reminder.'' The \texttt{<data>}/\texttt{</data>}
        tokens matter more than ``don't follow injected instructions.''
  \item \textbf{StruQ}: Reserved tokens for channel boundaries ($<$2\% ASR
        on manual attacks). Front-end filters ensure untrusted data cannot
        contain delimiter strings.
\end{itemize}

\noindent\textbf{Implication for us:} The Llama 3.1 chat template
\emph{already encodes} a natural privilege hierarchy through its role headers:
\texttt{<|start\_header\_id|>system<|end\_header\_id|>} (highest) $>$
\texttt{user} $>$ \texttt{tool} (lowest) $>$
\texttt{assistant} (model's decision).
We do not need custom boundary tokens (BIPIA) or reserved delimiters (StruQ).
The native template is the boundary marker. This is optimal because:
(a)~the model already has pre-trained awareness of these headers from
instruction tuning, and (b)~at inference time the agent framework sends
messages in exactly this format---zero distribution shift.

\paragraph{3. Forget/retain sets must be representationally distinct.}
Latent Unlearning's critical finding: when forget and retain distributions
share $>$20\% unigram overlap, representation-level unlearning \emph{fails
entirely}. Our harmful and benign tool calls share \emph{identical} JSON
format (\texttt{\{``name'': ``search\_web'', ``parameters'': ...\}}).

\medskip\noindent
\textbf{Implication:} This is exactly why \texttt{injection\_aware} works
and \texttt{cb\_full\_sequence} is suboptimal. \texttt{cb\_full\_sequence}
puts weight 1.0 on both the injection text AND the shared-format tool call
JSON. The model cannot separate ``harmful tool call'' from ``benign tool call''
when the surface tokens are nearly identical. \texttt{injection\_aware}
solves this by putting 0.5 weight on the injection span (which is
representationally distinct---HTML comments vs.\ clean queries) and 1.0
weight on post-injection assistant tokens (where the \emph{decision} diverges,
not the format).

\paragraph{4. Tool selection and argument generation are separate tasks.}
AgentFlux's DualTune finding: separating tool-name loss from argument loss
yields +22.5pp over joint fine-tuning. Hammer's finding: masking function
names to random strings forces description-based reasoning and improves
cross-benchmark generalization.

\medskip\noindent
\textbf{Implication:} For circuit-breaking, we primarily care about the
tool \emph{selection} decision (calling \texttt{search\_web} instead of
\texttt{retrieve\_multimodal\_docs}), not the arguments. Our
\texttt{tool\_name\_weighted} policy (implemented but untested in sweep)
captures exactly this insight: weight 1.0 on tool name tokens, 0.3 on
arguments. This could be combined with \texttt{injection\_aware} in future.

\paragraph{5. Multi-turn agentic format is necessary for deployment.}
The Instruction Hierarchy paper showed that training on injection in the
tool-output position of a multi-turn trace \emph{generalizes} to unseen
tools. Single-turn format (CB paper's default) loses the critical
\texttt{tool}$\to$\texttt{assistant} causal chain that makes agent injection
dangerous. At inference, the agent framework sends multi-turn messages;
training on the same format eliminates distribution shift.

\paragraph{6. Full-sequence loss is directionally better for agents.}
The prompt masking blog found +3pp pass rate when computing loss over the
full sequence (including prior tool-observation pairs) vs.\ completion-only.
Not statistically significant (p=0.10), but directionally consistent with
the CB paper's choice to apply loss to all tokens. Our \texttt{injection\_aware}
is a middle ground: it zeros out system/tool tokens but includes user
(injection) and assistant tokens---more selective than full-sequence, more
inclusive than completion-only.

\subsection{Conclusion: The Native Llama 3.1 Agentic Format Is Optimal}

No paper proposes a format that dominates for all three of our requirements
(representation rerouting + injection defense + tool-calling preservation).
Synthesizing the evidence, the optimal format is:

\begin{center}
\fbox{\parbox{0.92\textwidth}{
\textbf{Format:} Native Llama 3.1 multi-turn chat template with
\texttt{<|python\_tag|>} tool calls. \\
\textbf{Loss:} \texttt{injection\_aware} (0.5 on injection, 1.0 on
post-injection assistant, 0.0 elsewhere). \\
\textbf{Pool:} \texttt{cb\_full\_sequence} (broad representation sampling). \\
\textbf{Turns:} 5 messages (system $\to$ user $\to$ assistant[tool\_call]
$\to$ tool $\to$ assistant[follow-up]).
}}
\end{center}

\noindent
The exact rendered format for a harmful completed trace:

\begin{verbatim}
<|begin_of_text|><|start_header_id|>system<|end_header_id|>
                                                     w=0.0
{system prompt with tool schemas}<|eot_id|>
                                                     w=0.0
<|start_header_id|>user<|end_header_id|>
                                                     w=0.0
{clean query}                                        w=0.0
{injection text}                                     w=0.5
<|eot_id|>

                                                     w=1.0
<|python_tag|>{"name":"search_web","parameters":{..}}
                                                     w=1.0
<|eom_id|>

                                                     w=0.0
{tool output}<|eot_id|>
                                                     w=0.0
<|start_header_id|>assistant<|end_header_id|>
                                                     w=1.0
{follow-up response}<|eot_id|>                       w=1.0
\end{verbatim}

\noindent This is optimal because:

\begin{enumerate}
  \item \textbf{Native token distribution.} The same model generates these
        tokens during vLLM completion, so layer 10+20 representations are
        in-distribution. No cross-model artifacts (BIPIA used GPT-4, CB used
        an uncensored LLM).

  \item \textbf{Built-in privilege hierarchy.} The role headers
        (\texttt{system} $>$ \texttt{user} $>$ \texttt{tool}) serve the same
        function as BIPIA's \texttt{<data>} tokens and StruQ's
        \texttt{[MARK]} delimiters, but they are already in the model's
        vocabulary and pre-trained distribution.

  \item \textbf{Representational distinctness where it matters.} The 0.5
        weight goes on injection tokens (HTML comments, \texttt{<INFORMATION>}
        tags), which are surface-form distinct from clean queries. The 1.0
        weight goes on assistant decision tokens, where harmful (wrong tool)
        and benign (correct tool) diverge at the \emph{tool name} tokens.
        This addresses the n-gram overlap problem from latent unlearning.

  \item \textbf{Deployment-compatible.} At inference, the agent framework
        (LangChain, vLLM tool mode, etc.) sends messages in exactly this
        Llama 3.1 chat format. The representations the model sees during
        inference are identical to training. No format adaptation needed.

  \item \textbf{The \texttt{<|python\_tag|>} is a natural circuit-breaker
        target.} This special token precedes every tool call. In harmful
        traces, the layer 10+20 representation at this token position
        encodes ``I am about to call a tool that the injection told me to
        call.'' Rerouting this representation is precisely what CB needs.
\end{enumerate}

\section{Training Inefficiency Analysis (Mar 9, 2026)}
\label{sec:inefficiency}
% ===================================================================

Investigating why Sweep~9 deltas were only 2--5\% despite 1000+ steps and
confirmed representation-level signal, we identified \textbf{four compounding
inefficiencies} that collectively killed gradient flow through most of training.

\subsection{Bug 1: Alpha Decay Kills Push at Training End}

The coefficient schedule is:
\begin{equation}
  c_s = \alpha_{\max} \cdot \max\!\left(0,\; 1 - \frac{t}{\texttt{decay\_mult} \cdot T}\right)
\end{equation}
With \texttt{alpha\_decay\_multiplier}$=1.0$ (the default), $c_s = 0$ at
$t = T$. The harmful push is literally zero at the end of training.

\textbf{Fix:} Set \texttt{alpha\_decay\_multiplier}$=2.0$. At $t = T$,
$c_s = 0.5 \cdot \alpha_{\max}$ --- push remains at 50\% throughout training.

\subsection{Bug 2: Margin Saturates Instantly}

The hinge loss for the harmful term is:
\begin{equation}
  \mathcal{L}_{\text{harmful}} = \mathrm{ReLU}\!\left(m_h - d(\mathbf{h}_{\text{lora}},\, \mathbf{h}_{\text{frozen}})\right)
\end{equation}
With \texttt{margin\_harmful}$=1.6$ and \texttt{dl2rc} distance on
$\mathbb{R}^{4096}$ representations, the L2 norm alone easily exceeds 1.6
within 10--50 gradient steps. Once $d > m_h$, $\nabla \mathcal{L}_{\text{harmful}} = 0$.
\textbf{The harmful loss had zero gradient for 95--99\% of training.}

\textbf{Fix:} Set \texttt{margin\_harmful}$=20.0$. With dl2rc (L2 + 10$\cdot$ReLU(1$-$cos)),
this margin is large enough to sustain gradient flow through all of training.

\subsection{Bug 3: KL Fights the Push}

$\gamma_\mathrm{kl} = 0.3$ was held constant while $c_s$ decayed to 0. In the
second half of training, the KL divergence term (which penalizes deviation from
base model logits) was the dominant force, \emph{actively undoing} the harmful push.

\textbf{Fix:} Schedule KL with the retain coefficient:
\begin{equation}
  \gamma_{\text{kl,eff}} = \frac{c_r}{\alpha_{\max}} \cdot \gamma_\mathrm{kl}
\end{equation}
Since $c_r$ ramps from 0 to $\alpha_{\max}$, KL starts at zero (no interference
with harmful push) and gradually increases as retain becomes the priority.

\subsection{Bug 4: LR Decays on Top of Alpha}

\texttt{get\_linear\_schedule\_with\_warmup} decays LR to 0 at $t = T$.
Combined with $c_s \to 0$ (Bug~1), the effective learning signal was
proportional to:
\begin{equation}
  \text{effective} \propto \mathrm{LR}(t) \cdot c_s(t) \propto (1-t/T)^2
\end{equation}
A \textbf{quadratic decay}: at the midpoint of training, effective learning was
already at 25\% of initial. By 75\%, it was 6.25\%.

\textbf{Fix:} Use \texttt{get\_constant\_schedule\_with\_warmup} --- LR holds
constant after warmup. Combined with \texttt{decay\_mult}$=2.0$, effective
learning at end of training is $0.5 \cdot \alpha_{\max} \cdot \mathrm{LR}_0$.

\subsection{Combined Effect}

All four bugs are multiplicative. With all four present (Sweep~9):
\begin{equation}
  \text{effective}_{\text{sweep9}}(t=0.5T) = \mathrm{LR}_0 \cdot 0.5 \cdot \alpha_{\max} \cdot 0.5 \cdot \underbrace{\mathbb{1}[d < 1.6]}_{\approx 0} = \approx 0
\end{equation}
With all four fixed (Sweep~10):
\begin{equation}
  \text{effective}_{\text{sweep10}}(t=0.5T) = \mathrm{LR}_0 \cdot \alpha_{\max} \cdot 0.75 \cdot \mathbb{1}[d < 20.0] > 0
\end{equation}

\subsection{Synthesis: The Selectivity Problem}
\label{sec:selectivity}

The three tests reveal a fundamental tension:

\begin{enumerate}
  \item \textbf{Strong push direction} (Test~2): The model's representations
        at the generation point linearly separate ``comply'' from ``refuse''
        with AUC~0.96. CB can learn which direction to push.

  \item \textbf{Weak selectivity signal} (Test~3): At the same generation
        point, the model only WEAKLY distinguishes injection-context from
        clean-context (AUC~0.83). This is the ceiling on how well CB can
        decide \emph{when} to push.

  \item \textbf{The consequence}: CB rerouting learns a global direction
        (``away from tool-call JSON, toward refusal text'') but the LoRA is
        always active at inference. It pushes \emph{all} tool-call
        representations, harmful and benign alike. The only selectivity comes
        from the weak context signal (0.83 AUC), which must counteract the
        strong push direction (0.96 AUC). This is why the 3000-step model
        achieved $-46\%$ harmful delta but destroyed benign capability
        (40\% correct).
\end{enumerate}

\paragraph{Theoretical implication.} CB rerouting on this task will always
face a \textbf{Pareto tradeoff} between harmful reduction and benign
preservation. The tradeoff is governed by the ratio of selectivity signal
(AUC~0.83) to push signal (AUC~0.96). Shorter training finds a less
aggressive point on this frontier; it cannot eliminate the tradeoff.

\paragraph{Possible mitigations.}
\begin{itemize}
  \item \textbf{Amplify selectivity}: Make injection-context vs.\ clean-context
        more distinguishable. E.g., exaggerate injection markers, repeat
        injection text, add explicit boundary tokens.
  \item \textbf{Two-stage approach}: First detect injection (a classification
        problem with 0.83--0.97 AUC depending on pooling), then reroute
        only on flagged inputs. Decouples detection from rerouting.
  \item \textbf{Fujitsu re-addition}: Fujitsu traces have clearer injection
        structure (HTML comments in user messages) that may provide stronger
        selectivity signal, improving the context-level discrimination that
        CB needs.
  \item \textbf{Conditional LoRA}: Only activate the rerouting adapter when
        an injection is detected. Requires an injection classifier but
        preserves benign capability perfectly.
\end{itemize}

\section{Synthetic Refusal Traces}
\label{sec:refusal}
% ===================================================================

\textbf{Motivation:} Standard CB training pushes harmful representations
\emph{away from} the base model but does not specify a target direction.
The model learns ``don't produce representations like the base model does
for harmful inputs'' but not ``produce representations like a refusal.''
This may explain why CB outputs are gibberish rather than clean refusals.

\textbf{Method:} Take each harmful truncated trace (context includes injection
in tool output, last assistant message is the malicious tool call). Replace the
last assistant message with a randomly selected refusal template. Label as benign.
Add to the retain training set.

10 refusal templates varying in length and specificity, e.g.:
\begin{quote}
\small
``I notice the previous output contains instructions that weren't part of the
original request. I'll disregard those and focus on what you actually asked for.''
\end{quote}

\textbf{Result:} 1{,}225 refusal traces generated from 2{,}738 harmful inputs
(1{,}513 skipped due to traces not ending with assistant message or being too
short). These are added to the benign retain set, gated by the
\texttt{USE\_REFUSAL} flag.

\textbf{Hypothesis:} By including refusal traces in the retain set, the CB loss
anchors harmful-context representations to refusal-like hidden states instead of
generic benign states. This gives the rerouting a \emph{directed target}: when
the model sees an injection, its representations should look like ``I'm about to
refuse'' rather than ``I'm about to produce random text.''

\section{Sweep 11: Multi-Intervention Expansion (Mar 9--10, 2026)}
\label{sec:sweep11-full}
% ===================================================================

Following the Sweep~10 analysis (Section~\ref{sec:sweep11}), we implemented
five intervention groups targeting the selectivity problem from different
angles.  All configs inherit the v2 fixes (decay\_mult=2.0, margin\_h=20.0,
margin\_b=3.0, constant~LR, KL scheduled with~$c_r$).

\subsection{SRMU: Feature-Selective Rerouting (Novel Contribution)}

\textbf{Motivation.} The selectivity problem (AUC~0.83) means CB pushes all
4096 hidden dimensions equally.  Most dimensions are irrelevant to injection
detection, so pushing them disrupts benign representations for no gain.
Inspired by SRMU (Selective Representation Misdirection for Unlearning,
Jin~et~al.\ 2024), we apply an importance mask derived from a linear probe
trained on harmful-vs-benign hidden states.

\textbf{Implementation.}
\begin{enumerate}
  \item \textbf{Probe extraction} (\texttt{scripts/extract\_probe\_direction.py}):
        Train logistic regression on layer-10, last-token hidden states from
        the truncated AD traces.  Extract weight vector $\mathbf{w} \in
        \mathbb{R}^{4096}$.  Importance mask: $m_i = |w_i| / \max_j |w_j|$,
        so $m \in [0,1]^{4096}$.

  \item \textbf{Masked loss} (\texttt{src/training/losses.py}): Before
        computing dl2rc distance, apply element-wise masking:
        \[
          d = \mathrm{dl2rc}(\mathbf{h}_{\text{lora}} \odot \mathbf{m},\;
                             \mathbf{h}_{\text{frozen}} \odot \mathbf{m})
        \]
        Dimensions with small $|w_i|$ contribute negligibly to $d$, so the
        optimizer concentrates weight updates on the $\sim$20--30\% of
        dimensions that discriminate harmful from benign.

  \item \textbf{Pipeline integration} (\texttt{sweep\_ad\_focused.sbatch}):
        Step~0b auto-extracts the probe direction on the first SRMU job
        (requires GPU forward pass through base model).  The \texttt{.pt}
        file is cached at \texttt{configs/probe\_directions/layer10\_last\_token.pt}
        and reused by subsequent SRMU jobs.
\end{enumerate}

\textbf{Configs:}
\begin{center}
\small
\begin{tabular}{llll}
\toprule
Config & Steps & Data & Mask \\

\subsection{Balanced Configs (Fujitsu + AD)}

\textbf{Motivation.} Fujitsu CB outputs are clean correct tool calls (zero
gibberish across all Sweep~10 runs).  Adding Fujitsu data provides:
(1)~structural injection signal (HTML comments) that may transfer to AD,
(2)~diverse benign data (Fujitsu DR traces) to strengthen capability anchoring.

\textbf{Configs:} \texttt{ad\_500\_bal}, \texttt{ad\_750\_bal},

-27\%$ (best in Sweep~9) but \emph{hurt} AD ($+5\%$).  The v2 fixes may

\subsection{Balanced + Refusal}

\textbf{Config:} \texttt{ad\_750\_bal\_r} --- balanced data plus 1225 synthetic
refusal traces added to the benign retain set.

\textbf{Hypothesis:} The gibberish problem is the central quality issue.
Adding refusal traces gives CB a directed target: when the model sees injection
context, its representations should approach refusal-like states rather than
degenerate regions.  This addresses both gibberish and benign preservation.

\subsection{Alpha/Margin Tuning}

\textbf{ad\_1000\_a5} ($\alpha_{\max}=5.0$): Halving alpha slows Phase~1 push
(expect 60--100 steps to saturate margin instead of 30), reducing Phase~3
destructiveness.  Lower $c_s$ ceiling means less collateral damage to benign.

\textbf{ad\_1000\_m40} ($m_h=40.0$, $m_b=5.0$): Higher margin means harmful
reps must drift further to re-enter the active zone during Phase~2, potentially
delaying or preventing Phase~3 entirely.

\section{Lessons Learned (Cumulative)}
% ===================================================================

\begin{enumerate}
  \item \textbf{Cosine similarity + LoRA = dead at init.} LoRA weights start
        near zero, so $h_{\text{lora}} \approx h_{\text{frozen}}$ and
        $\cos(h_{\text{lora}}, h_{\text{frozen}}) \approx 1.0$. The ReLU in
        $\mathrm{ReLU}(\cos\!\operatorname{sim})$ clips this to 1.0 with gradient
        that points the wrong way. Require L2 or mixed distance.

  \item \textbf{Margins must be calibrated to the distance metric.} Margin 1.6
        with dl2rc on $\mathbb{R}^{4096}$ saturates in 10--50 steps.
        $\mathrm{ReLU}(m - d) = 0$ once $d > m$ means zero gradient.
        Margin should be set relative to the expected distance scale.

  \item \textbf{Multiple decaying schedules compound.} LR decay $\times$ alpha
        decay $\propto (1-t)^2$. Each schedule looks reasonable alone; together
        they create quadratic effective-learning decay. Prefer constant LR when
        using coefficient schedules.

  \item \textbf{KL divergence fights representation rerouting.} Constant KL
        penalty actively undoes the harmful push as the push coefficient decays.
        Schedule KL to ramp with the retain coefficient.

  \item \textbf{Data pipeline bugs are silent.} Zero-mask coverage, wrong tool
        schemas, missing benign files, schema mismatches --- none of these
        produce errors. Training loss still converges. Only eval reveals the problem.

  \item \textbf{``Training loss converges'' $\neq$ ``model learned anything.''}
        Always evaluate. A model can show declining loss while learning nothing
        useful (or learning to produce gibberish).

  \item \textbf{Probe separability with different contexts is trivially high.}
        Comparing traces with vs.\ without injection text will always show high
        AUC (0.97 mean-pooled). Same-context different-response (harmful vs.\
        refusal) is the meaningful test --- but even there, 0.96 last\_token AUC
        partly reflects surface token type (JSON vs.\ English), not pure
        behavioral intent.

  \item \textbf{The selectivity ceiling.} CB rerouting's effectiveness is
        bounded by the model's ability to distinguish injection-context from
        clean-context at the generation point (AUC~0.83, WEAK). The push
        direction is strong (0.96) but unselective. This predicts a Pareto
        tradeoff: more harmful reduction $\Rightarrow$ more benign destruction.

  \item \textbf{CB produces gibberish, not refusals.} Without a directed target
        (refusal representation), the model learns to produce degenerate token
        sequences that avoid the malicious tool call but are not useful text.
        Synthetic refusal data in the retain set may fix this by anchoring the
        rerouting target.

  \item \textbf{Verify against known-good baselines before exploring new configs.}
        Many early experiments were invalidated by bugs. Establish a verified
        baseline and re-run when bugs are found rather than layering fixes.

  \item \textbf{Three-phase training dynamic.} CB training with margin hinge loss
        follows Push $\to$ Plateau $\to$ Collapse. Phase~1 saturates the margin in
        $\sim$30 steps (zero harmful gradient thereafter). Phase~2 is benign-only
        training. Phase~3 begins at $\sim$68--72\% of training when representation
        drift pulls harmful distance back within margin, re-activating harmful
        push at the expense of benign. Stop before Phase~3 or strengthen benign
        anchoring to survive it.

  \item \textbf{More Phase~2 epochs $>$ fewer total steps.} ad\_5000\_v2 (72\%
        benign) outperformed ad\_3000\_v2 (40\% benign) despite longer training
        because it had 3470 steps of benign anchoring vs 2010. The ratio of
        Phase~2 to Phase~3 duration matters more than absolute step count.

  \item \textbf{Injection structure determines CB output quality.} Structural
        injections (Fujitsu HTML comments, 1-to-1 tool mapping) allow clean
        redirect outputs. Semantic injections (AD \texttt{<INFORMATION>} tags,
        diverse goals, 64 tools) produce gibberish because no single clear
        alternative exists in representation space.

  \item \textbf{Pareto dominance requires multi-metric evaluation.} Looking at
        AD harmful reduction alone would pick ad\_3000\_v2 ($-46\%$). Adding
        benign and Fujitsu metrics reveals ad\_5000\_v2 dominates on the Pareto
        frontier. Always evaluate all metrics before declaring a best config.
\end{enumerate}
